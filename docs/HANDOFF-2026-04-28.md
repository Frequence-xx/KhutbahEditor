## Known issues + suggested next work

These were flagged by the final code reviewer; none are correctness-blocking.

**Reviewer's "Minor" list (deferred follow-ups):**
- ID collision risk: `proj-${Date.now()}` used in 3 Shell handlers. Replace with crypto.randomUUID().
- `REVIEW_THRESHOLD = 0.9` duplicated in `JobManager.ts` and `ReviewPane.tsx`. Extract to a shared module.
- `Bridge.auth` type narrower than the actual `window.khutbah.auth` surface. Widen or stop reaching around it.
- Settings `computeDevice` change requires app restart — surface this in SettingsPane.
- Z-index collision: modal + toast both `z-50`. Toast should sit above modal (or pick separate z layers).
- `cleanup()` import inconsistency across 4 test files — the `tests/renderer/setup.ts` globalizes it, but a few files still import + call locally. Prune.

**Real bugs / open work:**
- **Stale projects from before the GUI strip**: persisted v0 → v1 migrated projects keep `runState='ready'` (or `'idle'`) but have no `part1`/`part2`. ReviewPane renders empty tabs. Workaround: hover-× delete. Proper fix: add an "idle/no-data" right-pane with [Run detection] + [Delete] buttons.
- **YouTube ingest path**: `Shell.handleSubmitYoutube` calls `jm.startDetect(id)` with the URL stuffed into `sourcePath`. The sidecar's `detect.run` will receive a URL where it expects a file path → fail. Need to chain `ingest.youtube_download` before detect.
- **Spec §6 auto-pilot**: `src/lib/autopilot.ts` was deleted as orphaned. When the auto-upload-on-high-confidence behaviour is ready to ship, design from scratch against the new JobManager contract — don't resurrect.
- **Project deletion mid-job**: `JobManager.cancel(id)` is called by Shell.handleDelete, but stale entries in `progressState` / `inFlight` for a deleted project would orphan if cancel races. Worth a defensive sweep.

## Hot spots when touching code

- `src/store/projects.ts` — extending `Project` requires updating the migration in the same file. Persisted state in localStorage will retain the old shape until users open the app.
- `src/jobs/JobManager.ts` — three async entry points (`startDetect`, `fireCut`, `runUpload`) all subscribe to `bridge.onProgress` and unsubscribe in `.finally()`. Adding a fourth needs the same pattern.
- `python-pipeline/khutbah_pipeline/__main__.py` — every RPC method is registered here. Adding a new method requires touching this + updating the renderer-side IPC contract.
- `tailwind.config.js` — the brand palette is the single source of truth. Don't use `bg-slate-*`, `bg-amber-400`, etc. Use `bg-bg-1`, `bg-amber`, `text-text-strong`, etc.
